# 报告视觉规范

## 视觉系统

参考 kami 风格中文长报告：

- **背景**：暖纸色 `#f7f3ec`
- **主文字**：墨黑 `#1a1a1a`
- **强调色**：墨蓝 `#1e3a5f`
- **辅助色**：朱砂 `#a83b3b`、苔绿 `#5a7a3a`、赭黄 `#b8860b`
- **字体**：`-apple-system, "PingFang SC", "Noto Serif SC", serif`
- **正文行高**：1.8
- **图表标题**：18px，加粗，居左

## ECharts 通用配置

- 容器宽度：100%，最小高度 360px
- 移动端断点：768px 以下单列
- 高密度图表必须监听 resize 重绘
- 横向条形图标签折行：`axisLabel.width: 120, overflow: 'truncate'`
- 词云使用 `echarts-wordcloud` 插件 CDN

## 内容原则

- 不编造数据，缺失字段在图表角注明「数据不全」
- 报告底部标注「本报告基于本人豆瓣账号数据生成，仅供个人复盘使用」
- 不嵌入任何 cookie / session 信息
